✅ Important Points (Write in Theory Section)
1. Database Design Concept
The system follows a Relational Database Model.
Data is organized into multiple related tables to avoid redundancy.
2. Entity Explanation
Student → Stores student details.
Instructor → Stores faculty details.
Course → Stores course information.
Enrollment → Acts as a junction table.
3. Relationship Explanation
Student ↔ Course → Many-to-Many
Implemented using Enrollment table.
Instructor → Course → One-to-Many
One instructor teaches multiple courses.
4. Keys Used
Primary Key
Uniquely identifies each record.
Example: StudentID, InstructorID, CourseID, EnrollmentID.
Foreign Key
Maintains relationship between tables.
Ensures referential integrity.
5. Referential Integrity
Prevents invalid data entry.
Example:
Cannot insert a Course with non-existing InstructorID.
Cannot enroll a student who doesn’t exist.
6. Normalization
Database is in 3rd Normal Form (3NF):
No redundancy
No partial dependency
No transitive dependency
7. Data Types Used
INT → Numeric identifiers
VARCHAR → Text fields
DATE → Enrollment tracking
8. Advantages of This Design
Reduces duplication
Improves data consistency
Easy to query using JOINs
Scalable for real systems
