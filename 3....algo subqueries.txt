✅ Important Points (Theory Section)
🔹 1. WHERE Clause (Filtering)
Used to extract specific records based on condition
Example: Age > 20
🔹 2. ORDER BY Clause (Sorting)
Used to arrange data:
ASC → Ascending (default)
DESC → Descending
🔹 3. GROUP BY Clause
Groups rows having same values
Used with aggregate functions
🔹 4. Aggregate Functions
COUNT() → counts rows
AVG() → calculates average
Others: SUM(), MAX(), MIN()
🔹 5. Observation (Important Correction)
SELECT StudentID, AVG(Age) FROM Student GROUP BY StudentID;
Since StudentID is unique → each group has 1 row
👉 So AVG(Age) = same as Age

✔ Better conceptual query:

SELECT AVG(Age) FROM Student;
🔹 6. Nested Subquery
Query inside another query
Executes inner query first
🔹 7. Types of Subqueries
Single-row
Multi-row (IN, ANY, ALL)

✔ Your example uses IN (multi-row subquery)

🔹 8. Advantages of Subquery
Simplifies complex queries
Improves readability
Avoids multiple joins (in some cases)
🔹 9. Performance Consideration
Subqueries can be slower than JOINs for large data
Indexing improves performance
🔹 10. Execution Order of SQL Query
FROM
WHERE
GROUP BY
HAVING
SELECT
ORDER BY
🔹 11. HAVING vs WHERE
WHERE → filters before grouping
HAVING → filters after grouping

Example:

SELECT CourseID, COUNT(*)
FROM Enrollment
GROUP BY CourseID
HAVING COUNT(*) > 1;
🔹 12. NULL Handling
Aggregate functions ignore NULL values
🔹 13. Index Usage
Improves performance of:
WHERE
ORDER BY
GROUP BY
