✅ Important Points (Theory Section)
🔹 1. View Definition
A virtual table derived from SQL query
Does not store actual data
🔹 2. Types of Views
Simple View → Single table, supports UPDATE
Complex View → Multiple tables, mainly for reporting
🔹 3. Advantages of Views
Simplifies complex queries
Enhances security
Provides abstraction
Improves readability
🔹 4. View Limitations
Cannot always update complex views
Depends on base tables
May reduce performance for large queries
🔹 5. Updatable View Conditions
Based on single table
No JOIN or GROUP BY
Contains primary key
🔹 6. WITH CHECK OPTION
Ensures valid updates through view
Prevents violating view conditions
🔹 7. Index Definition
A structure that improves data retrieval speed
🔹 8. Index Internals
Uses B-Tree structure
Reduces search time:
Without index → O(n)
With index → O(log n)
🔹 9. When Index is Used
In WHERE, JOIN, ORDER BY clauses
🔹 10. When Index is NOT Used
Small tables
Using functions on column
Pattern starting with %
🔹 11. Primary Key and Index
Primary Key automatically creates index
No need to manually index it again
🔹 12. Composite Index
Index on multiple columns for faster queries
🔹 13. Performance Measurement
Use EXPLAIN to check query execution
🔹 14. Advantages of Index
Faster search
Better query performance
🔹 15. Disadvantages of Index
Extra memory required
Slows INSERT/UPDATE
🔹 16. Security Using Views
Restrict access to sensitive columns
🔹 17. DDL and DML
DDL → CREATE, DROP, ALTER
DML → INSERT, UPDATE, SELECT
🔹 18. Real-Life Applications
Student systems
Banking
E-commerce
🔹 19. Best Practices
Index only frequently used columns
Avoid over-indexing
🔹 20. Exam-Level Observation
Views = Logical abstraction
Index = Physical optimization
